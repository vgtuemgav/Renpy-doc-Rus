=========================
Оптимизация языка экранов
=========================

Ren'Py использует ряд техник для оптимизации скорости языка экранов.
При использовании Ren'Py для создания сложных интерфейсов, например,
для симуляторов, полезно понимать, как работает язык экранов, чтобы
достичь максимальной производительности.

Это руководство применимо ко второй реализации языка экранов, которая
была добавлена в Ren'Py 6.18. Если ваша игра была создана в Ren'Py 6.17
или ранее, может потребоваться выбрать опцию "Force Recompile"
(Принудительная перекомпиляция) в лаунчере, чтобы обеспечить
обновление её экранов до последней версии.

Это руководство не заменяет хорошие практики программирования. Если
экран использует вложенные циклы для выполнения большого количества
непродуктивной работы, он будет медленнее, чем экран, который избегает
таких циклов. Хотя понимание техник из этого руководства важно, полное
избегание лишней работы всегда лучше, чем позволять Ren'Py
оптимизировать работу за вас.

Список параметров
=================

Для лучшей производительности все экраны должны быть определены со списком
параметров. Когда экран не принимает параметров, он должен быть определен
с пустым списком параметров. Этот экран::

    screen test():
        vbox:
            for i in range(10):
                text "[i]"

быстрее, чем этот::

    screen test:
        vbox:
            for i in range(10):
                text "[i]"

Когда экран определяется без списка параметров, любое имя, используемое
в этом экране, может быть переопределено при его показе. Это требует от
Ren'Py быть более консервативным при анализе экрана, что может
ограничить выполняемую оптимизацию.

Предсказание
============

Экраны работают лучше, когда они предсказаны заранее. Это потому,
что Ren'Py выполнит код экрана во время предсказания и загрузит
изображения, используемые экраном.

Существует два способа, которыми Ren'Py автоматически предсказывает экраны:

* Ren'Py будет предсказывать экраны, показанные операторами
  ``show screen`` и ``call screen``.
* Ren'Py будет предсказывать экраны, которые будут показаны
  действиями :func:`Show` и :func:`ShowMenu`.

Если экраны показываются из Python, хорошей практикой является начать
предсказание экрана до того, как он будет показан. Чтобы начать
предсказание экрана, используйте функцию
:func:`renpy.start_predict_screen`. Чтобы остановить предсказание
экрана, используйте функцию :func:`renpy.stop_predict_screen`.


Повторное использование отображаемых объектов
=============================================

При вычислении оператора языка экранов, который создает отображаемый
объект (displayable), Ren'Py проверяет, равны ли позиционные аргументы
и свойства, переданные этому объекту, позиционным аргументам и
свойствам, переданным при последнем вычислении этого оператора.
Если они равны, вместо создания нового отображаемого объекта, Ren'Py
повторно использует существующий.

Повторное использование отображаемых объектов имеет ряд преимуществ для
производительности. Это экономит затраты на создание нового объекта,
что может быть значительно для объектов с большим количеством
внутренних состояний. Что еще важнее, повторное использование
отображаемого объекта означает, что во многих случаях Ren'Py не нужно
будет повторно отрисовывать объект перед его показом пользователю, что
может привести к еще одному значительному ускорению.

Для сравнения позиционных аргументов и свойств Ren'Py использует понятие
равенства, воплощенное оператором == в Python. Мы расширили это понятие
равенства на действия, решив, что два действия должны быть равны, когда
они неотличимы друг от друга — когда не имеет значения, какое действие
вызывается или какое действие опрашивается для определения
чувствительности (sensitivity) или выбранности (selectedness).

Все действия, поставляемые с Ren'Py, соответствуют этому определению.
При определении собственных действий имеет смысл обеспечить их этим
понятием равенства. Это можно сделать, предоставив соответствующий
метод ``__eq__``. Например::

    class TargetShip(Action):
        def __init__(self, ship):
            self.ship = ship

        def __eq__(self, other):
            if not isinstance(other, TargetShip):
                return False

            return self.ship is other.ship

        def __call__(self):
            global target
            target = self.ship

Важно определять функцию ``__eq__`` аккуратно, убедившись, что она
сравнивает все поля и использует сравнение по значению (==) и по
ссылке (is) соответствующим образом.

Константные выражения и чистые функции
======================================

Ren'Py может использовать свойства константных переменных и чистых
функций для повышения скорости вычисления экранов и полного исключения
вычисления некоторых их частей.

Определения
-----------

Выражение является **const** (сокращение от constant — константа),
если оно всегда представляет одно и то же значение при вычислении.
Для целей Ren'Py выражение является константным тогда и только тогда,
когда следующие выражения всегда вычисляются в одно и то же
константное значение или не определены:

* Применение любого унарного, бинарного или тернарного оператора
  к выражению, при условии, что другие операнды также являются
  константными.
* Доступ к полю выражения.
* Индексирование выражения, будь то с помощью числа или объекта.

Числа и строки Python являются константными, так же как и литералы
list, tuple, set и dict, у которых все компоненты константны. Ren'Py
помечает переменные, определенные с помощью оператора ``define``, как
константные. Функции :func:`renpy.const` и :func:`renpy.not_const`
могут быть использованы для дальнейшего контроля того, что Ren'Py
считает константным. Список константных имен по умолчанию приведен
в разделе :ref:`Const Names <const-names>` ниже.

Если у вас есть переменная, которая никогда не будет меняться, имеет
смысл использовать ``define`` как для ее определения, так и для
объявления ее константной. Например::

    define GRID_WIDTH = 20
    define GRID_HEIGHT = 10

Вызываемая функция, класс или действие являются **чистыми** (pure),
если при передаче константных значений в качестве всех своих аргументов
они всегда возвращают одно и то же константное значение.
Альтернативно, выражение, вызывающее чистую функцию с константными
аргументами, также является константным выражением.

Большое количество стандартных функций, классов и действий помечены
как чистые. Эти функции перечислены в разделе
:ref:`Pure Names <pure-names>` ниже.

Функции объявляются чистыми с помощью функции :func:`renpy.pure`,
которую можно использовать как декоратор для функций, объявленных
в хранилище по умолчанию (default store).

Константные выражения и чистые функции не обязаны сохранять одно и то
же значение после следующих событий:

* Конец фазы init.
* Смена языка.
* Перестройка стилей.

Как константы оптимизируют язык экранов
---------------------------------------

Существует три преимущества в обеспечении константности аргументов и
свойств языка экранов.

Во-первых, константные аргументы и свойства вычисляются при подготовке
экранов, то есть в конце фазы init, при смене языка или при
перестройке стилей. После этого больше не нужно тратить время на
вычисление константных аргументов и свойств.

Во-вторых, константы хорошо сочетаются с повторным использованием
отображаемых объектов. Когда все аргументы и свойства отображаемого
объекта константны, его всегда можно повторно использовать, что дает
все преимущества такого подхода.

Наконец, когда Ren'Py встречает дерево отображаемых объектов, у
которого все аргументы, свойства и выражения, влияющие на поток
управления, также константны, Ren'Py повторно использует все дерево,
не вычисляя выражений и не создавая отображаемых объектов. Это может
дать значительный прирост производительности.

Например, следующий экран не выполняет никакого Python-кода и не
создает никаких отображаемых объектов после первого его предсказания
или показа::

    screen mood_picker():
        hbox:
            xalign 1.0
            yalign 0.0

            textbutton "Happy" action SetVariable("mood", "happy")
            textbutton "Sad" action SetVariable("mood", "sad")
            textbutton "Angry" action SetVariable("mood", "angry")

Константный текст
-----------------

При определении текста обратите внимание, что строки, содержащие
подстановки текста нового стиля, являются константными::

    $ t = "Hello, world."
    text "[t]"

Прямая передача переменной, содержащей текст, как правило, не является
константной::

    $ t = "Hello, world."
    text t

То же самое касается использования процентной подстановки::

    $ t = "Hello, world."
    text "%s" % t

Наконец, обратите внимание, что функция перевода текста _ является
чистой, поэтому если она содержит строку, все выражение является
константным::

    text _("Your score is: [score]")

Если переменная, содержащая текст, содержит подстановку, необходимо
использовать флаг преобразования ``!i``::

    $ who = "Jane"
    $ t = "Hello, [who]!"
    text 'Then I told her, "[t!i]"'
