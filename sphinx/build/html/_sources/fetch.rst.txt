Запросы HTTPS/HTTP
==================

Ren'Py поддерживает получение информации по HTTP и HTTPS с помощью функции
`renpy.fetch`. Эта функция:

* Поддерживает GET, POST и PUT-запросы.
* Поддерживает отправку на сервер данных или json-объектов через POST или PUT.
* Может возвращать результат в виде байтов, строки или json-объекта.

Функция `fetch` предназначена для использования в основном потоке Ren'Py после запуска игры.
Её можно использовать на десктопах, мобильных устройствах и на веб-платформе.
При использовании на веб-платформе, если запрос идёт не на тот же сервер, который
изначально обслуживал игру, сервер должен поддерживать `CORS <https://developer.mozilla.org/ru/docs/Web/HTTP/CORS>`_.

В качестве очень простого примера, вот так можно получить новости с сервера::

    $ news = renpy.fetch("https://example.com/news.txt", result="text")
    "Вот новости: [news]"

А вот так можно отправить JSON на сервер и получить JSON в ответ::

    $ result = renpy.fetch("https://example.com/api", json={"name": "Ren'Py"}, result="json")

Как и в любом приложении, которое взаимодействует по сети, при использовании `renpy.fetch`
необходимо учитывать вопросы безопасности, особенно при отображении полученных
таким образом медиафайлов. (Ren'Py в целом не защищён от вредоносных изображений,
видео- и аудиофайлов.)

Requests
--------

На десктопах и мобильных устройствах Ren'Py включает библиотеку `Requests <https://requests.readthedocs.io/>`_.
Она более мощная, но интегрируется не так хорошо, как `renpy.fetch`.
(Например, Ren'Py вызовет :func:`renpy.pause` для ожидания данных, в то время как
`requests` заблокирует выполнение, что может привести к таким проблемам, как прерывание звука.)
