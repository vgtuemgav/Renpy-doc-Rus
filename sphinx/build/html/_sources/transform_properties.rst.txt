======================
Свойства трансформации
======================

Свойства трансформации используются :doc:`transforms` для влияния на то, как отображается displayable. Обычно они задаются с помощью :ref:`ATL <atl>` или класса :class:`Transform`.

.. _transform-properties:

Список свойств трансформации
============================

Существуют следующие свойства трансформации.

Когда в качестве типа указана :term:`position`, её относительная компонента интерпретируется как доля от размера содержащей области (для :tpref:`pos`) или от размера отображаемого объекта (displayable) (для :tpref:`anchor`).

Обратите внимание, что не все свойства независимы. Например, :tpref:`xalign` и :tpref:`xpos` оба обновляют одни и те же внутренние данные. В инструкции ``parallel`` не более одного блока должно изменять свойства, использующие общие данные. Свойства угла и радиуса устанавливают как горизонтальное, так и вертикальное положение.

Позиционирование
----------------

.. transform-property:: pos

    :type: (position, position)
    :default: (0, 0)

    Позиция относительно верхнего левого угла содержащей области.

.. transform-property:: xpos

    :type: position
    :default: 0

    Горизонтальная позиция относительно левого края содержащей области.

.. transform-property:: ypos

    :type: position
    :default: 0

    Вертикальная позиция относительно верхнего края содержащей области.

.. transform-property:: anchor

    :type: (position, position)
    :default: (0, 0)

    Позиция якоря относительно верхнего левого угла отображаемого объекта (displayable).

.. transform-property:: xanchor

    :type: position
    :default: 0

    Горизонтальная позиция якоря относительно левого края отображаемого объекта (displayable).

.. transform-property:: yanchor

    :type: position
    :default: 0

    Вертикальная позиция якоря относительно верхнего края отображаемого объекта (displayable).

.. transform-property:: align

    :type: (float, float)
    :default: (0.0, 0.0)

    Эквивалентно установке одинаковых значений для pos и anchor.

.. transform-property:: xalign

    :type: float
    :default: 0.0

    Эквивалентно установке этого значения для xpos и xanchor.

.. transform-property:: yalign

    :type: float
    :default: 0.0

    Эквивалентно установке этого значения для ypos и yanchor.

.. transform-property:: offset

    :type: (absolute, absolute)
    :default: (0, 0)

    Количество пикселей, на которое смещается отображаемый объект (displayable) в каждом направлении. Положительные значения смещают вправо-вниз.

.. transform-property:: xoffset

    :type: absolute
    :default: 0

    Количество пикселей, на которое смещается отображаемый объект (displayable) в горизонтальном направлении. Положительные значения смещают вправо.

.. transform-property:: yoffset

    :type: absolute
    :default: 0

    Количество пикселей, на которое смещается отображаемый объект (displayable) в вертикальном направлении. Положительные значения смещают вниз.

.. transform-property:: xycenter

    :type: (position, position)
    :default: (0.0, 0.0)

    Эквивалентно установке pos в значение этого свойства, а anchor в (0.5, 0.5).

.. transform-property:: xcenter

    :type: position
    :default: 0.0

    Эквивалентно установке xpos в значение этого свойства, а xanchor в 0.5.

.. transform-property:: ycenter

    :type: position
    :default: 0.0

    Эквивалентно установке ypos в значение этого свойства, а yanchor в 0.5.

.. transform-property:: subpixel

    :type: boolean
    :default: False

    Если True, дочерний элемент будет размещён с использованием субпиксельного позиционирования.

    Субпиксельное позиционирование влияет на цвета (включая прозрачность), которые отрисовываются в пикселях, но не на то, какие пиксели отрисовываются. Когда субпиксельное позиционирование используется в сочетании с движением (что является обычным случаем), изображение должно иметь прозрачные границы в тех направлениях, в которых оно может двигаться, если эти края видны на экране.

    Например, если спрайт персонажа движется горизонтально, имеет смысл сделать прозрачные границы слева и справа. Они могут не понадобиться при панорамировании фона, выходящего за пределы видимой области, так как края не будут видны.

Вращение
--------

.. transform-property:: rotate

    :type: float or None
    :default: None

    Если None, вращение не происходит. В противном случае, изображение будет повёрнуто на указанное количество градусов по часовой стрелке. Вращение отображаемого объекта (displayable) приводит к изменению его размера в соответствии с настройкой rotate_pad, описанной ниже. Это может вызвать изменение позиционирования, если xanchor и yanchor не равны 0.5.

.. transform-property:: rotate_pad

    :type: boolean
    :default: True

    Если True, то повёрнутый отображаемый объект (displayable) дополняется отступами так, чтобы его ширина и высота были равны гипотенузе исходных ширины и высоты. Это гарантирует, что трансформация не изменит размер при вращении её содержимого. Если False, трансформация получит минимальный размер, который вмещает преобразованный displayable. Это больше подходит для фиксированных вращений.

.. transform-property:: transform_anchor

   :type: boolean
   :default: False

   Если true, точка якоря располагается на обрезанном дочернем элементе и масштабируется и вращается вместе с ним. Фактически, это делает якорь точкой, вокруг которой вращается и масштабируется дочерний элемент.

Масштабирование и отражение
---------------------------

.. transform-property:: zoom

    :type: float
    :default: 1.0

    Вызывает масштабирование отображаемого объекта (displayable) на указанный коэффициент.

.. transform-property:: xzoom

    :type: float
    :default: 1.0

    Вызывает горизонтальное масштабирование отображаемого объекта (displayable) на указанный коэффициент. Отрицательное значение вызывает горизонтальное отражение изображения.

.. transform-property:: yzoom

   :type: float
   :default: 1.0

   Вызывает вертикальное масштабирование отображаемого объекта (displayable) на указанный коэффициент. Отрицательное значение вызывает вертикальное отражение изображения.

Пиксельные эффекты
------------------

.. transform-property:: nearest

    :type: boolean
    :default: None

    Если True, отображаемый объект (displayable) и его дочерние элементы отрисовываются с использованием фильтрации по ближайшему соседу. Если False, они отрисовываются с использованием билинейной фильтрации. Если None, значение наследуется от родительского элемента или берётся из :var:`config.nearest_neighbor`, которое по умолчанию равно False.

.. transform-property:: alpha

    :type: float
    :default: 1.0

    Управляет непрозрачностью отображаемого объекта (displayable).

    Трансформация альфа-канала применяется к каждому изображению, составляющему дочерний элемент трансформации, независимо. Это может привести к неожиданным результатам, когда дочерние элементы перекрываются, например, к просвечиванию персонажа сквозь одежду. Displayable :func:`Flatten` может помочь в решении этих проблем.

.. transform-property:: additive

    :type: float
    :default: 0.0

    Управляет степенью аддитивного смешивания, выполняемого Ren'Py. При значении 1.0 Ren'Py рисует с использованием оператора ADD. При 0.0 Ren'Py рисует с использованием оператора OVER.

    Аддитивное смешивание выполняется для каждого дочернего элемента трансформации независимо.

    Полностью аддитивное смешивание не изменяет альфа-канал назначения, и аддитивные изображения могут быть не видны, если они не нарисованы непосредственно на непрозрачной поверхности. (Сложные операции, такие как viewport, :func:`Flatten`, :func:`Frame` и некоторые переходы, могут вызывать проблемы с аддитивным смешиванием.)

.. transform-property:: matrixcolor

    :type: None or Matrix or MatrixColor
    :default: None

    Если не None, значение этого свойства используется для перекрашивания всего, что рисуют дочерние элементы этой трансформации. Интерполяция поддерживается только при использовании MatrixColors, и если эти MatrixColors структурно схожи. Смотрите :doc:`matrixcolor` для получения дополнительной информации.

.. transform-property:: blur

    :type: None or float
    :default: None

    Размывает дочерний элемент этой трансформации на `blur` пикселей, вплоть до границы displayable. Точные детали размытия могут меняться между версиями Ren'Py, и размытие может проявлять артефакты, особенно когда размываемое изображение меняется.

    Размытие вызывает "сплющивание" (flatten) своего дочернего элемента, при котором он отрисовывается на прозрачном фоне перед применением размытия.

Полярное позиционирование
-------------------------

.. transform-property:: around

    :type: (position, position)
    :default: (0.0, 0.0)

    Определяет начальную точку относительно верхнего левого угла содержащей области, из которой будет проведён полярный вектор (вычисленный из :tpref:`angle` и :tpref:`radius`). Сумма этих двух значений даёт итоговое свойство :tpref:`pos`.

.. transform-property:: angle

    :type: float

    Задаёт угловую компоненту позиции, указанной в полярных координатах.
    Измеряется в градусах, где 0 соответствует верху экрана, а 90 — правому краю.

    Ren'Py ограничивает этот угол в диапазоне от 0 до 360 градусов, включая 0, но не включая 360. Если установлено значение вне этого диапазона, оно будет приведено к эквивалентному углу в этом диапазоне перед использованием. (Установка значения -10 эквивалентна установке 350.)

.. transform-property:: radius

    :type: position

    Радиальная компонента позиции, заданной в полярных координатах.

    Если это float, значение будет масштабировано до меньшего из ширины и высоты, доступных трансформации.

Полярное позиционирование якоря
-------------------------------

.. note::

    Хотя использовать полярные координаты для позиционирования якоря возможно, часто удобнее просто установить :tpref:`anchor` в (0.5, 0.5) и позиционировать центр вашего displayable.

.. transform-property:: anchoraround

    :type: (position, position)

    Определяет начальную точку относительно верхнего левого угла displayable, из которой будет проведён полярный вектор (вычисленный из :tpref:`anchorangle` и :tpref:`anchorradius`). Сумма этих двух значений даёт итоговое свойство :tpref:`anchor`.

.. transform-property:: anchorangle

    :type: (float)

    Угловая компонента полярных координат якоря. Указывается в градусах, где 0 — верх, а 90 — право.

    Ren'Py ограничивает этот угол в диапазоне от 0 до 360 градусов, включая 0, но не включая 360. Если установлено значение вне этого диапазона, оно будет приведено к эквивалентному углу в этом диапазоне перед использованием. (Установка значения -10 эквивалентна установке 350.)

.. transform-property:: anchorradius

    :type: (position)

    Радиальная компонента полярных координат якоря.

    Если это float, значение масштабируется горизонтально и вертикально под размер и форму displayable: если высота не равна ширине, радиус, который не является строго абсолютным, приведёт к эллиптическому движению при изменении anchorangle. По этой причине рекомендуется передавать этому свойству только значения типа ``int`` или :func:`absolute`.

Обрезка и изменение размера
---------------------------

.. transform-property:: crop

    :type: None or (position, position, position, position)
    :default: None

    Если не None, вызывает обрезку displayable до указанного прямоугольника. Прямоугольник задаётся как кортеж (x, y, width, height), где x и y — это координаты верхнего левого угла прямоугольника относительно верхнего левого угла дочернего элемента. Все значения могут выходить за границы исходного изображения, при этом область снаружи становится прозрачной, однако width и height должны быть положительными.

    Если заданы и corners, и crop, crop имеет приоритет.

.. transform-property:: corner1

    :type: None or (position, position)
    :default: None

    Если не None, задаёт верхний левый угол области обрезки. Значения могут выходить за границы исходного изображения. Crop имеет приоритет над corners.

.. transform-property:: corner2

    :type: None or (position, position)
    :default: None

    Если не None, задаёт правый нижний угол области обрезки. Значения могут выходить за границы исходного изображения, но не должны быть меньше :tpref:`corner1`. Crop имеет приоритет над corners.

.. transform-property:: xysize

    :type: None or (position, position)
    :default: None

    Если не None, вызывает масштабирование displayable до указанного размера. Это эквивалентно установке свойств :tpref:`xsize` и :tpref:`ysize` в первую и вторую компоненты соответственно.

    На это влияет свойство :tpref:`fit`.

.. transform-property:: xsize

    :type: None or position
    :default: None

    Если не None, вызывает масштабирование displayable до указанной ширины.

    На это влияет свойство :tpref:`fit`.

.. transform-property:: ysize

    :type: None or position
    :default: None

    Если не None, вызывает масштабирование displayable до указанной высоты.

    На это влияет свойство :tpref:`fit`.

.. transform-property:: fit

    :type: None or string
    :default: None

    Приводит к изменению размера displayable в соответствии с таблицей ниже. В контексте этой таблицы "размерами" являются:

    * Если и :tpref:`xsize`, и :tpref:`ysize` не равны None, оба размера используются как размеры.
    * Если только одно из этих свойств не равно None, оно используется как единственный размер.
    * В противном случае, если fit не равно None, в качестве размеров используется область, в которой содержится Transform.

    Если fit, xsize и ysize все равны None, это свойство не применяется.

    .. list-table::
       :widths: 15 85
       :header-rows: 1

       * - Значение
         - Описание
       * - ``contain``
         - Максимально большой, не выходя за рамки размеров. Сохраняет соотношение сторон.
       * - ``cover``
         - Минимально возможный, но соответствующий всем размерам или превосходящий их. Сохраняет соотношение сторон.
       * - None или ``fill``
         - Растягивает/сжимает displayable для точного соответствия размерам.
       * - ``scale-down``
         - Как ``contain``, но никогда не увеличит размер displayable.
       * - ``scale-up``
         - Как ``cover``, но никогда не уменьшит размер displayable.

Панорамирование и замощение
---------------------------

.. transform-property:: xpan

    :type: None or float
    :default: None

    Если не None, интерпретируется как угол, который используется для горизонтального панорамирования по панорамному изображению с обзором 360 градусов. Центр изображения используется как нулевой угол, а левый и правый края — как -180 и 180 градусов соответственно.

.. transform-property:: ypan

    :type: None or float
    :default: None

    Если не None, интерпретируется как угол, который используется для вертикального панорамирования по панорамному изображению с обзором 360 градусов. Центр изображения используется как нулевой угол, а верхний и нижний края — как -180 и 180 градусов соответственно.

.. transform-property:: xtile

    :type: int
    :default: 1

    Количество раз для замощения изображения по горизонтали.

.. transform-property:: ytile

    :type: int
    :default: 1

    Количество раз для замощения изображения по вертикали.

Переходы
--------

Смотрите :ref:`atl-transitions`.

.. transform-property:: delay

    :type: float
    :default: 0.0

    Если эта трансформация используется как переход, это его длительность. Смотрите :ref:`atl-transitions`.

.. transform-property:: events

    :type: boolean
    :default: True

    Если True, события передаются дочернему элементу этой трансформации. Если False, события блокируются. (Это можно использовать в ATL-переходах, чтобы предотвратить попадание событий в old_widget.)

Прочее
------

.. transform-property:: fps

    :type: float or None
    :default: None

    Если не None, это изменяет время внутри трансформации, делая его дискретным.
    Например, если у трансформации fps равен 10, то время внутри трансформации будет округлено в меньшую сторону до ближайшего кратного 0.1. Это можно использовать для симуляции более низкой частоты кадров.

.. transform-property:: show_cancels_hide

    :type: boolean
    :default: True

    Обычно, когда отображаемый объект или экран с тем же тегом или именем, что и скрываемый, показывается, скрываемый объект или экран удаляется, отменяя трансформацию hide. Если это свойство установлено в False в трансформации hide, отмена не произойдёт, и трансформация hide завершится полностью.

Есть также несколько наборов свойств трансформации, которые документированы в других местах:

Свойства 3D-сцены:
    :tpref:`perspective`, :tpref:`point_to`, :tpref:`orientation`, :tpref:`xrotate`, :tpref:`yrotate`, :tpref:`zrotate`, :tpref:`matrixanchor`, :tpref:`matrixtransform`, :tpref:`zpos`, :tpref:`zzoom`

Свойства рендеринга на основе моделей:
    :tpref:`blend`, :tpref:`mesh`, :tpref:`mesh_pad`, :tpref:`shader`

GL-свойства:
    :ref:`GL-свойства <gl-properties>`.

Uniform-переменные:
    Свойства, начинающиеся с ``u_``, являются uniform-переменными, которые могут использоваться :ref:`пользовательскими шейдерами <custom-shaders>`.

Порядок применения свойств
==========================

Свойства трансформации применяются в следующем порядке:

#. fps
#. mesh, blur
#. tile
#. pan
#. crop, corner1, corner2
#. xysize, size, maxsize
#. zoom, xzoom, yzoom
#. point_to
#. orientation
#. xrotate, yrotate, zrotate
#. rotate
#. zpos
#. matrixtransform, matrixanchor
#. zzoom
#. perspective
#. nearest, blend, alpha, additive, shader
#. matrixcolor
#. GL-свойства, Uniform-переменные
#. свойства позиционирования
#. show_cancels_hide


Устаревшие свойства трансформации
=================================

.. warning::

    Следующие свойства не следует использовать в современных играх, так как они могут конфликтовать с новыми функциями. Они оставлены здесь только для совместимости, вместе с новым способом достижения того же поведения.

.. transform-property:: alignaround

    :type: (float, float)

    Устанавливает одинаковое значение для :tpref:`anchor`, :tpref:`around` и :tpref:`anchoraround`.

.. transform-property:: crop_relative

    :type: boolean
    :default: True

    Если False, компоненты типа float в :tpref:`crop`, :tpref:`corner1` и
    :tpref:`corner2` интерпретируются как абсолютное число пикселей, а не как
    доля от ширины и высоты исходного изображения.

    Для выражения абсолютного числа пикселей следует передавать этим свойствам
    экземпляры :func:`absolute`, а не использовать свойство crop_relative.
    При необходимости, значения сомнительного типа можно обернуть в вызов :func:`absolute`.

.. transform-property:: size

    :type: None or (int, int)
    :default: None

    Это старая версия :tpref:`xysize`, интерпретирующая значения с плавающей точкой как абсолютное число пикселей.

.. transform-property:: maxsize

    :type: None or (int, int)
    :default: None

    Если не None, вызывает масштабирование displayable так, чтобы он вписался в рамку этого размера с сохранением соотношения сторон. (Обратите внимание, что это означает, что один из размеров может быть меньше размера этой рамки.)

    Для достижения того же результата передайте значения свойству :tpref:`xysize` и
    установите свойство :tpref:`fit` в значение "contain".
