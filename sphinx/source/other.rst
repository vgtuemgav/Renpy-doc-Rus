===========================
Прочие функции и переменные
===========================

На этой странице перечислены и документированы различные прочие функции и переменные,
которые не подходят для размещения в других местах.


Версия Ren'Py
-------------

.. include:: inc/renpy_version

.. var:: renpy.version_string

    Номер версии Ren'Py в виде строки формата "Ren'Py 1.2.3.456".

.. var:: renpy.version_only

    Номер версии Ren'Py без префикса "Ren'Py". Строка формата "1.2.3.456".

.. var:: renpy.version_tuple

    Номер версии Ren'Py в виде кортежа (tuple) формата (1, 2, 3, 456).

    Это именованный кортеж (namedtuple) с четырьмя полями: ``major``, ``minor``, ``patch``,
    и ``commit``.

.. var:: renpy.version_name

    Читаемое человеком название версии, вида "Example Version.".

.. var:: renpy.license

    Строка, содержащая текст лицензии, который должен быть включен
    в экран "О программе" (about screen).

Определение платформы
---------------------

Ren'Py включает в себя ряд переменных, которые устанавливаются в зависимости от того, на какой платформе
он работает.

.. var:: renpy.windows

    Имеет истинное значение при запуске в Windows.

.. var:: renpy.macintosh

    Имеет истинное значение при запуске в macOS.

.. var:: renpy.linux

    Имеет истинное значение при запуске в Linux или других POSIX-подобных операционных системах.

.. var:: renpy.android

    Имеет истинное значение при запуске на Android.

.. var:: renpy.ios

    Имеет истинное значение при запуске на iOS.

.. var:: renpy.emscripten

    Имеет истинное значение при запуске в браузере. Это значение, в частности,
    модуль emscripten, который позволяет вам взаимодействовать с браузером, см. :func:`renpy.emscripten.run_script`
    и подобные функции.

.. var:: renpy.mobile

    Имеет истинное значение при запуске на Android, iOS или в браузере.

Эти переменные устанавливаются только при запуске на реальных устройствах, а не
в эмуляторах. Они больше предназначены для использования в специфичном для платформы
коде на Python. Для компоновки отображения используйте :ref:`варианты экранов <screen-variants>`.

renpy.random
------------

Этот объект является генератором случайных чисел, который реализует
`интерфейс генерации случайных чисел Python <https://docs.python.org/library/random.html>`_.
Случайность может быть сгенерирована путем вызова различных методов, которые
предоставляет этот объект. Полный список см. в документации Python, но наиболее полезными
являются:

* ``renpy.random.random()``

Возвращает следующее случайное число с плавающей точкой в диапазоне (0.0, 1.0).

* ``renpy.random.randint(a, b)``

Возвращает случайное целое число N такое, что a <= N <= b.

* ``renpy.random.choice(seq)``

Возвращает случайный элемент из непустой последовательности `seq`.

* ``renpy.random.shuffle(seq)``

Перемешивает элементы последовательности `seq` на месте. Это не возвращает
список, а изменяет существующий.

В отличие от стандартного генератора случайных чисел Python, этот объект взаимодействует с
откатом (rollback), генерируя те же числа независимо от того, сколько раз мы откатываемся.
Его следует использовать вместо стандартного модуля random Python. ::

    # вернуть случайное число с плавающей точкой от 0 до 1
    $ randfloat = renpy.random.random()

    # вернуть случайное целое число от 1 до 20
    $ d20roll = renpy.random.randint(1, 20)

    # вернуть случайный элемент из списка
    $ randfruit = renpy.random.choice(['apple', 'orange', 'plum'])

* ``renpy.random.Random(seed=None)``

Возвращает новый объект генератора случайных чисел, отдельный от основного, инициализированный
указанным значением, если оно предоставлено.

SDL
---

Доступ к DLL SDL2 можно получить с помощью :func:`renpy.get_sdl_dll`.
Это позволяет использовать функции SDL2 напрямую. Однако их использование часто
требует знания модуля ctypes Python.

Нет никаких гарантий относительно версии SDL2, включенной
в Ren'Py, включая то, какие функции будут или не будут скомпилированы. Эти
функции могут не работать на платформах, которые в остальном могут запускать Ren'Py.
Важно проверить на значение None перед продолжением.

В следующем примере позиция окна будет взята из SDL2::

    init python:

        import ctypes

        def get_window_position():
            """Retrieves the position of the window from SDL2.

            Returns the (x, y) of the upper left corner of the window, or
            (0, 0) if it's not known.
            """

            sdl = renpy.get_sdl_dll()

            if sdl is None:
                return (0, 0)

            win = renpy.get_sdl_window_pointer()

            if win is None:
                return (0, 0)

            x = ctypes.c_int()
            y = ctypes.c_int()

            result = sdl.SDL_GetWindowPosition(win, ctypes.byref(x), ctypes.byref(y))

            return result
