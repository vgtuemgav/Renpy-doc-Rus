HTTPS/HTTP Updater
==================

В Ren'Py встроено средство обновления (updater), которое может автоматически
загружать и устанавливать обновления для игры Ren'Py с веб-хоста. Обновления
работают на десктопных платформах, за исключением приложений для macOS.

Во время этого процесса средство обновления Ren'Py показывает экран обновления,
предлагая пользователю продолжить и позволяя отменить операцию при
необходимости.

Поддерживаются два формата обновлений. Современный формат называется rpu
и был представлен в Ren'Py 8.2. Старый формат, называемый zsync, устарел,
но может быть сгенерирован для обновления со старых версий.

Требования к серверу
--------------------

Средство обновления требует, чтобы вы предоставили свой собственный хостинг.
Вы должны иметь возможность загружать файлы обновлений, переходя по
соответствующему URL напрямую, и ваш сервер должен поддерживать HTTP range
queries (запросы диапазонов).

Сборка обновления
-----------------

Обновления создаются автоматически при сборке дистрибутивов. Чтобы собрать
обновление, установите :var:`build.include_update` в True в options.rpy. Это
разблокирует опцию "Build Updates" в разделе "Build Distributions"
лаунчера. Отметьте эту опцию, и Ren'Py создаст файлы обновления.

Файлы обновления состоят из:

updates.json
   Индекс доступных обновлений и их версий.

updates.ecdsa
   Подпись файла updates.json, используемая для проверки того, что он не
   был изменён.

rpu/
   Каталог rpu содержит метаданные и данные, используемые для обновлений.

Вы должны загрузить их в одно и то же место на вашем веб-сервере.

-------

Чтобы запустить обновление, вызовите либо updater.update, либо

Экран
-----

Чтобы настроить внешний вид средства обновления, вы можете переопределить экран ``updater``.
Вот пример такого экрана::

   screen updater(u=None):

      add "#000"

      frame:
         style_group ""

         has side "t c b":
            spacing gui._scale(10)

         label _("Updater")

         fixed:

            vbox:

               if u.state == u.ERROR:
                  text _("Произошла ошибка:")
               elif u.state == u.CHECKING:
                  text _("Проверка обновлений.")
               elif u.state == u.UPDATE_NOT_AVAILABLE:
                  text _("Программа обновлена до последней версии.")
               elif u.state == u.UPDATE_AVAILABLE:
                  text _("[u.version] доступна. Вы хотите её установить?")
               elif u.state == u.PREPARING:
                  text _("Подготовка к загрузке обновлений.")
               elif u.state == u.DOWNLOADING:
                  text _("Загрузка обновлений.")
               elif u.state == u.UNPACKING:
                  text _("Распаковка обновлений.")
               elif u.state == u.FINISHING:
                  text _("Завершение.")
               elif u.state == u.DONE:
                  text _("Обновления установлены. Программа будет перезапущена.")
               elif u.state == u.DONE_NO_RESTART:
                  text _("Обновления установлены.")
               elif u.state == u.CANCELLED:
                  text _("Обновления были отменены.")

               if u.message is not None:
                  null height gui._scale(10)
                  text "[u.message!q]"

               if u.progress is not None:
                  null height gui._scale(10)
                  bar value (u.progress or 0.0) range 1.0 style "_bar"

         hbox:

            spacing gui._scale(25)

            if u.can_proceed:
               textbutton _("Продолжить") action u.proceed

            if u.can_cancel:
               textbutton _("Отмена") action u.cancel

Экрану обновления передаётся один параметр, объект Updater, который
должен называться `u`. Объект Updater имеет следующие поля, которые
можно использовать для настройки экрана:

.. class:: updater.Updater

   .. attribute:: state

      Текущее состояние средства обновления. Возможные значения и их смысл
      смотрите в примере выше. Все значения являются константами
      объекта Updater.

   .. attribute:: message

      Если не None, сообщение для отображения пользователю.

   .. attribute:: progress

      Если не None, прогресс текущей операции в виде числа с плавающей запятой
      от 0.0 до 1.0.

   .. attribute:: can_proceed

      Если True, экран должен отобразить кнопку, которая позволит
      пользователю продолжить обновление.

   .. attribute:: proceed

      Если can_proceed имеет значение True, это действие, которое должно быть
      вызвано, когда пользователь нажимает кнопку продолжения.

   .. attribute:: can_cancel

      Если True, экран должен отобразить кнопку, которая позволит
      пользователю отменить обновление.

   .. attribute:: cancel

      Если can_cancel имеет значение True, это действие, которое должно быть
      вызвано, когда пользователь нажимает кнопку отмены.

   .. attribute-:: old_disk_total

      Если не None, целое число, указывающее общее количество байтов на диске,
      которое игра занимала в начале обновления.

   .. attribute-:: new_disk_total

      Если не None, целое число, указывающее общее количество байтов на диске,
      которое игра будет занимать в конце обновления.

   .. attribute-:: download_total

      Если не None, целое число, указывающее общее количество байтов, которые
      будут загружены во время обновления.

   .. attribute-:: download_done

      Если не None, целое число, указывающее количество байтов, которые уже
      были загружены во время обновления.

   .. attribute-:: write_total

      Если не None, целое число, указывающее общее количество байтов, которые
      будут записаны на диск во время обновления.

   .. attribute-:: write_done

      Если не None, целое число, указывающее количество байтов, которые уже
      были записаны на диск во время обновления.
