Экранные действия, значения и функции

Ren'Py поставляется с рядом действий, значений и функций, предназначенных для использования с экранами и экранным языком.

Действия

Действия (actions) вызываются, когда кнопка (включая imagebuttons, textbuttons и hotspots) активируется, на неё наводится курсор или курсор убирается с неё. Действия могут определять, когда кнопка является выбранной или неактивной.

Помимо этих действий, действием может быть функция, не принимающая аргументов. Функция вызывается, когда инициируется действие. Если действие возвращает значение, то это значение возвращается из взаимодействия (interaction).

Вместо одного действия обычно можно предоставить список действий, и в этом случае действия из списка выполняются по порядку. Список действий является активным, если все действия в нём активны, и выбранным, если хотя бы одно из них выбрано; если в списке нет SensitiveIf() или SelectedIf() соответственно.

Управляющие действия

Это действия, которые управляют экранами, результатами взаимодействия и потоком управления.

Call(label, *args, **kwargs)

Завершает текущую инструкцию и вызывает метку label, заданную в виде строки. Позиционные и именованные аргументы передаются в renpy.call().

Hide(screen=None, transition=None, _layer=None, immediately=False)

Это действие скрывает экран, если он показан.

screen

Строка с именем экрана, который нужно скрыть, или None, чтобы скрыть текущий экран.

transition

Если не None, переход, который происходит при скрытии экрана.

_layer

Передаётся в качестве аргумента layer в renpy.hide_screen(). Игнорируется, если screen равно None.

immediately

Если True, экран скрывается немедленно, без события 'on hide'.

Jump(label)

Передаёт управление на метку label, заданную в виде строки.

NullAction()

Ничего не делает.

Можно использовать, чтобы кнопка реагировала на события наведения/снятия курсора, но при этом ничего не делала.

Return(value=None)

Вызывает возврат из текущего взаимодействия с указанным ненулевым значением. Часто используется с меню и imagemaps для выбора возвращаемого значения взаимодействия. Если экран был вызван с помощью инструкции call screen, возвращаемое значение помещается в переменную _return.

Находясь в меню, это действие возвращает из меню. (В этом случае значение должно быть None.)

Show(screen, transition=None, *args, **kwargs)

Показывает другой экран. screen — это строка с именем экрана. Аргументы передаются показываемому экрану.

Если не None, transition используется для показа нового экрана.

Это действие принимает именованные аргументы _layer, _zorder и _tag, которые имеют то же значение, что и в функции renpy.show_screen().

ShowTransient(screen, transition=None, *args, **kwargs)

Показывает временный экран (transient). Временный экран будет скрыт по завершении текущего взаимодействия. Аргументы передаются показываемому экрану.

Если не None, transition используется для показа нового экрана.

Это действие принимает именованные аргументы _layer, _zorder и _tag, которые имеют то же значение, что и в функции renpy.show_screen().

ToggleScreen(screen, transition=None, *args, **kwargs)

Переключает видимость экрана screen. Если он не показан, он показывается с предоставленными аргументами. В противном случае экран скрывается.

Если не None, transition используется для показа и скрытия экрана.

Это действие принимает именованные аргументы _layer, _zorder и _tag, которые имеют то же значение, что и в функции renpy.show_screen().

Акцессоры определяют цель, значение которой будет изменено, а управляющий определяет, каким будет новое значение. Их поведение относительно просто для понимания:

  • Действия -Variable изменяют значение глобальной переменной с именем name, находящейся в общем хранилище. Аргумент name должен быть строкой и может быть простым именем, таким как "strength", или именем с точками, разделяющими переменную и поля, например "hero.strength" или "persistent.show_cutscenes".

  • Действия -ScreenVariable изменяют значение переменной с именем name, связанной с текущим экраном верхнего уровня. В экране, используемом через use, это действие устанавливает переменную в контексте экрана, содержащего все используемые экраны.

  • Действия -LocalVariable изменяют значение переменной с именем name, локальной для экрана, в котором она находится. Это действие полезно только в экране, который был использован другим экраном через use (для получения дополнительной информации см. Use). Во всех остальных случаях следует предпочесть действия -ScreenVariable, так как они обеспечивают лучшую производительность и позволяют кэшировать большую часть экрана. Действия -LocalVariable должны быть созданы в контексте, в котором устанавливается переменная — их нельзя передать откуда-либо ещё.

  • Действия -Field изменяют значение поля с именем field объекта object.

  • Действия -Dict изменяют значение ключа key в словаре dict: они изменяют dict[key]. Это также работает со списками.

  • Действия Set- просто устанавливают значение цели равным переданному value. Обратите внимание, что это не имеет ничего общего со встроенным типом set в Python. target = value

  • Действия Toggle- инвертируют логическое значение своей цели между true_value (если задано и не None) и false_value (аналогично). Когда true_value и false_value оба равны None, target = not target

  • Действия Cycle- циклически перебирают предоставленные values, которые должны быть непустой последовательностью (списком, кортежем или диапазоном). Если значение цели отсутствует в последовательности на момент выполнения действия, оно устанавливается в первое значение в последовательности. Параметр loop (по умолчанию True) определяет, что происходит, когда values заканчиваются: если True, цикл начинается с начала, если False — вызывается исключение. Параметр reverse (по умолчанию False) переворачивает переданную последовательность values.

  • Действия Increment- добавляют amount к своей цели, по умолчанию равный 1, но может быть любого типа, совместимого с целью. target = target + amount

Следующие действия с данными не соответствуют вышеуказанному шаблону. Некоторые из них связаны с типом Python set, не путать с действиями Set- выше.

AddToSet(set, value)

Добавляет value в set.

set

Множество, в которое нужно добавить. Это может быть множество Python или список, в этом случае значение добавляется в конец списка.

value

Значение, которое нужно добавить.

RemoveFromSet(set, value)

Удаляет value из set.

set

Множество, из которого нужно удалить. Это может быть множество или список.

value

Значение, которое нужно удалить.

ToggleSetMembership(set, value)

Переключает членство value в set. Если значения нет в множестве, оно добавляется. В противном случае, оно удаляется.

Кнопки с этим действием отмечаются как выбранные тогда и только тогда, когда значение находится в множестве.

set

Множество, в которое нужно добавить или из которого удалить. Это может быть множество или список. В случае списка новые элементы добавляются в конец.

value

Значение, которое нужно добавить или удалить.

Действия меню

Эти действия вызывают меню или в основном полезны в главном или игровом меню.

Continue(regexp='[^_]', confirm=True)

Вызывает загрузку последнего сохранения. Предназначение этой функции — загрузить последнее сохранение игрока из главного меню.

regexp

Если присутствует, будет использоваться в renpy.newest_slot. Стандартный шаблон продолжит игру с любого сохранения, включая быстрые и автоматические сохранения. Если вы хотите продолжать только с сохранений, созданных игроком, установите это значение в r"\d".

confirm

Если true, Ren'Py спросит пользователя, хочет ли он продолжить с того места, где остановился, если он находится не в главном меню.

MainMenu(confirm=True, save=True)

Вызывает возврат Ren'Py в главное меню.

confirm

Если true, Ren'Py спросит пользователя, желает ли он вернуться в главное меню, вместо прямого возврата.

save

Если true, игра сохраняется в _quit_slot перед тем, как Ren'Py перезапустится и вернет пользователя в главное меню. Игра не сохраняется, если _quit_slot равен None.

Quit(confirm=None)

Выход из игры.

confirm

Если true, запрашивает у пользователя подтверждение на выход, вместо немедленного выхода. Если None, запрос появится только в том случае, если пользователь находится не в главном меню.

ShowMenu(screen=_game_menu_screen, *args, _transition=config.intra_transition, **kwargs)

Заставляет нас войти в игровое меню, если мы еще не там. Если мы находимся в игровом меню, то эта функция показывает экран или переходит к метке.

screen обычно является именем экрана, который отображается с помощью механизма экранов. Если экран не существует, к его имени добавляется "_screen", и осуществляется переход к этой метке. Ему передаются *args и **kwargs. Если экран не существует, к его имени добавляется "_screen", и осуществляется переход к этой метке, игнорируя args и kwargs.

Если задан необязательный именованный аргумент _transition, меню будет менять экраны, используя указанный переход. Если он не указан вручную, используется переход по умолчанию config.intra_transition.

  • ShowMenu("load")

  • ShowMenu("save")

  • ShowMenu("preferences")

Это также можно использовать для отображения пользовательских экранов меню. Например, если у вас определен экран "stats", его можно показать как часть игрового меню с помощью:

  • ShowMenu("stats")

Вызов ShowMenu без аргумента войдет в игровое меню на экране по умолчанию, взятом из _game_menu_screen.

Дополнительные позиционные и именованные аргументы передаются экрану.

Start(label='start')

Вызывает выход Ren'Py из контекста меню и переход к указанной метке. Основное применение — запуск новой игры из главного меню. Типичные примеры использования:

  • Start() - Начать с метки start.

  • Start("foo") - Начать с метки "foo".

Настройки (Preferences)

Хотя все настройки могут быть определены на основе действий и значений, приведенных выше, для выбора правильных требуется некоторое знание Ren'Py. Конструктор настроек упрощает это, создавая действие или значение в зависимости от имён, используемых в экране настроек по умолчанию.

Эта функция создаёт соответствующее действие или значение из настройки (preference). Имя настройки должно быть таким же, как в стандартных меню, а значение должно быть либо именем выбора, "toggle" для переключения между вариантами, конкретным значением, либо отсутствовать в случае кнопок.

Всплывающие подсказки (Tooltips)

Теперь к всплывающим подсказкам можно получить доступ через свойство tooltip, доступное для всех отображаемых объектов, и функцию GetTooltip. Функция GetTooltip возвращает значение свойства tooltip, когда отображаемый объект получает фокус.

Напоминаем, что значения, передаваемые в свойство tooltip, должны поддерживать операцию сравнения на равенство.

Вот пример:

screen tooltip_example():
    vbox:
        textbutton "North":
            action Return("n")
            tooltip "To meet a polar bear."

        textbutton "South":
            action Return("s")
            tooltip "All the way to the tropics."

        textbutton "East":
            action Return("e")
            tooltip "So we can embrace the dawn."

        textbutton "West":
            action Return("w")
            tooltip "Where to go to see the best sunsets."

        $ tooltip = GetTooltip()

        if tooltip:
            text "[tooltip]"

Отображаемый объект Nearrect можно использовать для отображения всплывающих подсказок в стиле "popup" и он поддерживает специальное имя фокуса "tooltip", которому присваивается местоположение последнего фокуса, установившего подсказку:

screen tooltip_example2():
    frame:

        padding (20, 20)
        align (.5, .3)

        has vbox

        textbutton "North":
            action Return("n")
            tooltip "To meet a polar bear."

        textbutton "South":
            action Return("s")
            tooltip "All the way to the tropics."

        textbutton "East":
            action Return("e")
            tooltip "So we can embrace the dawn."

        textbutton "West":
            action Return("w")
            tooltip "Where to go to see the best sunsets."

    # This has to be the last thing shown in the screen.

    $ tooltip = GetTooltip()

    if tooltip:

        nearrect:
            focus "tooltip"
            prefer_top True

            frame:
                xalign 0.5
                text tooltip
GetTooltip(screen=None, last=False)

Возвращает всплывающую подсказку (tooltip) для элемента в фокусе или None, если ни один элемент не находится в фокусе.

screen

Если не None, это должно быть имя или тег экрана. Если указано, эта функция возвращает подсказку, только если элемент в фокусе является частью этого экрана.

last

Если истинно, возвращает последнее не-None значение, которое эта функция могла бы вернуть.

Предупреждение

Этот способ был заменён вышеописанным, но вы можете встретить его в старых проектах.

Класс tooltip изменяет экран, когда курсор наводится на кнопку.

class Tooltip(default)

Объект всплывающей подсказки (tooltip) можно использовать для определения части экрана, которая обновляется, когда мышь наводится на определенную область.

Объект tooltip имеет поле value, которое устанавливается в значение default, переданное в конструктор при создании tooltip'а. Когда кнопка, использующая действие, созданное tooltip'ом, оказывается под курсором, поле value изменяется на значение, связанное с этим действием.

Action(value)

Возвращает действие, которое обычно используется как свойство hovered для кнопки. Когда курсор наведен на кнопку, поле value этого tooltip'а устанавливается в value. Когда кнопка теряет фокус, поле value этого tooltip'а возвращается к значению по умолчанию.

При использовании tooltip с экраном обычным поведением является создание объекта tooltip в инструкции default. Затем значение tooltip и метод action можно использовать внутри экрана. Порядок использования внутри экрана не имеет значения — можно использовать значение до того, как будет использовано действие.

Всплывающие подсказки могут принимать любое значение. Хотя в примере ниже мы используем инструкцию text для отображения строки на экране, также возможно использовать инструкцию add для добавления отображаемого объекта. Возможно и более сложное поведение.

screen tooltip_test:

    default tt = Tooltip("No button selected.")

    frame:
        xfill True

        has vbox

        textbutton "One.":
            action Return(1)
            hovered tt.Action("The loneliest number.")

        textbutton "Two.":
            action Return(2)
            hovered tt.Action("Is what it takes.")

        textbutton "Three.":
            action Return(3)
            hovered tt.Action("A crowd.")

        text tt.value
