Пространства имён
=================

Ren'Py использует пространства имён для организации переменных и методов. Этот файл содержит описания некоторых
пространств имён, которые использует Ren'Py, в основном для помощи редакторам в предоставлении автодополнения и документации.

.. var:: achievement

    Пространство имён, содержащее функции, которые предоставляют и управляют :doc:`достижениями <achievement>`. Оно также содержит большую часть
    интеграции со Steamworks.

.. var:: bubble:

    Пространство имён, содержащее переменные, которые управляют отображением :doc:`диалоговых пузырей <bubble>`.

.. var:: build:

    Пространство имён, содержащее переменные, которые управляют :doc:`процессом сборки <build>`.

.. var:: config:

    Пространство имён, содержащее переменные, которые управляют :doc:`конфигурацией <config>` Ren'Py. Эти переменные должны
    быть установлены во время инициализации (в блоках ``init python`` или с помощью ``define``) и не должны изменяться после запуска
    игры.

.. var:: define:

    Пространство имён `define` содержит функции, которые определяют новые переменные, такие как семейства переходов.

.. var:: director:

    Пространство имён `director` содержит функции, которые управляют :doc:`интерактивным режиссёром <director>`, который
    позволяет интерактивно вставлять изображения и музыку в игру.

.. var:: gui

    Пространство имён `gui` содержит функции, которые управляют :doc:`стандартной системой GUI <gui>`. Эти переменные имеют значение только
    в том случае, если вы используете стандартную систему GUI, и могут не использоваться, если вы её заменили. Инструкции `define`,
    влияющие на пространство имён `gui`, повторно выполняются при изменении перевода.

.. var:: iap

    Пространство имён `iap` содержит функции, которые управляют :doc:`системой внутриигровых покупок <iap>`.

.. var:: im

    **Примечание: Большинство функций в пространстве имён `im` устарели.**

    Пространство имён `im` содержит манипуляторы изображений, которые загружают или обрабатывают изображения на CPU. Большинство функций здесь
    можно выполнить на GPU с помощью :class:`Transform`.

.. var:: layeredimage

    :doc:`Слоистые изображения <layeredimage>` — это способ объединения нескольких изображений в одно, используя атрибуты
    и условия для управления тем, какие изображения показываются. Пространство имён `layeredimage` содержит классы, которые позволяют
    создавать и манипулировать слоистыми изображениями из Python, что эквивалентно инструкции ``layeredimage``.

.. var:: persistent

    Пространство имён `persistent` содержит :doc:`постоянные <persistent>` данные. Поля этого объекта изначально имеют значение None и сохраняют
    свои значения между запусками игры, даже если слот сохранения не загружен.

    Значения полей объекта `persistent` должны быть типами, предоставляемыми Python, такими как булевы значения, числа,
    строки, списки, кортежи, словари и множества. Классы, которые вы определяете, не должны присваиваться объекту `persistent`.

.. var:: preferences

    Пространство имён :doc:`preferences <preferences>` содержит переменные, которые содержат настройки. Хотя их можно читать и устанавливать,
    наиболее распространённое использование — с инструкцией ``default``, используя синтаксис вроде::

        default preferences.fullscreen = True

.. var:: preferences.volume

    Пространство имён :doc:`preferences <preferences>` содержит переменные, которые устанавливают громкость по умолчанию для каждого микшера. Их
    следует устанавливать с помощью инструкции ``default``, например::

        default preferences.volume.music = 0.5

.. var:: renpy

    Пространство имён `renpy` содержит функции и классы, которые являются частью самого движка Ren'Py. Они могут быть
    эквивалентны инструкциям языка Ren'Py или вводить функциональность, которая не заслуживает отдельной инструкции.

.. var:: renpy.audio.filter

    Пространство имён `renpy.audio.filter` содержит классы и функции, которые создают :doc:`аудиофильтры <audio_filters>`.

.. var:: renpy.music

    Пространство имён `renpy.audio.music` содержит функции, которые управляют :doc:`аудиосистемой <audio>`. Эти функции
    по умолчанию работают с музыкальным каналом.

.. var:: renpy.sound

    Пространство имён `renpy.audio.sound` содержит функции, которые управляют :doc:`аудиосистемой <audio>`. Эти функции
    по умолчанию работают со звуковым каналом. Большинство функций задокументированы под их эквивалентами в `renpy.music`.

.. var:: style

    Пространство имён `style` содержит стили и функции, которые управляют стилями. Стили используются для управления внешним видом
    текста, изображений и других элементов в Ren'Py. Использование инструкции `style` предпочтительнее прямого использования пространства имён
    `style`.

.. var:: ui

    **Примечание: Большинство функций в пространстве имён `ui` устарели.**

    Пространство имён `ui` содержит старые функции и классы, используемые для отображения элементов пользовательского интерфейса. За исключением
    `ui.adjustment`, `ui.interact`, `ui.callsinnewcontext` и `ui.invokesinnewcontext`, эти функции
    устарели.

.. var:: updater

    Пространство имён `updater` содержит функции, классы и переменные, которые управляют :doc:`обновлениями по HTTP/HTTPS <updater>`.
