Пользовательские операторы
==========================

Пользовательские операторы (Creator-Defined Statements, CDS) позволяют вам добавлять свои собственные операторы в скриптовый язык Ren'Py. Это даёт возможность добавлять функциональность, которая не поддерживается текущим синтаксисом.

CDS могут быть более гибкими, чем их эквивалент на чистом Python.

Например, случайный выбор реплики диалога::

    label introduction:
        python:
            greetings = ['Hello.', 'Welcome.', 'Can I help you?']
            greeting = renpy.random.choice(greetings)

        "[greeting]"


Парсер Ren'Py не знает заранее, что происходит в блоке python или как его следует выполнять.
Он ничего не делает с этим кодом до момента выполнения и вызывает ошибку, если возникает исключение.

Использование CDS позволяет:

- Проверять корректность разобранного синтаксиса (например, проверять, что элементы в списке, отправленном в renpy.random.choice, содержат корректный текст).

- Игнорировать некорректные данные при выполнении (для некритичных функций часто лучше пропустить выполнение, чем выбрасывать исключение).

- Предсказывать Displayable-объекты (если функция их использует).

- Предоставлять вам дополнительную информацию во время :ref:`lint` (если во время выполнения ошибка была проигнорирована, вы можете получить здесь отчёт).

Например, то же поведение, но написанное в виде CDS::

    python early:
        def parse_random(lexer):
            subblock_lexer = lexer.subblock_lexer()
            choices = []

            while subblock_lexer.advance():
                with subblock_lexer.catch_error():
                    statement = subblock_lexer.renpy_statement()
                    choices.append(statement)

            return choices


        def next_random(choices):
            return renpy.random.choice(choices)


        def lint_random(parsed_object):
            for i in parsed_object:
                    check = renpy.check_text_tags(i.block[0].what)
                    if check:
                        renpy.error(check)


        renpy.register_statement(
            name="random",
            block=True,
            parse=parse_random,
            next=next_random,
            lint=lint_random,
        )


``random`` теперь доступен как оператор::

    label introduction:
        random:
            "Hello."
            "Welcome."
            "Can I help you?"


Использование CDS не гарантирует успешного выполнения, но чем лучше вы напишете код своего оператора, тем лучше Ren'Py сможет "понять", чего вы от него ожидаете.


Использование
-------------

Пользовательские операторы (CDS) должны соответствовать следующим правилам:

- Они должны быть определены в блоке ``python early``.

- Их нельзя использовать в том же файле, в котором они определены.

- Файл, содержащий CDS, должен быть загружен раньше любого файла, который его использует.
  (Поскольку Ren'Py загружает файлы в порядке сортировки их путей по стандарту Юникод, обычно имеет смысл добавлять префикс, такой как 01 или другое небольшое число, к имени любого файла, содержащего CDS. См. :ref:`early-phase` для получения дополнительной информации о порядке загрузки файлов в Ren'Py, включая особенности для директорий game/libs/ и game/mods/.)

Пользовательские операторы регистрируются с помощью функции :func:`renpy.register_statement`. Эта функция принимает другие функции, которые выполняют операции над содержимым CDS.

Например, новый оператор с именем ``line``, который позволяет указывать строки текста без кавычек.

::

    line e "These quotes will show up," Eileen said, "and don't need to be backslashed."

Функции парсинга будет отправлено содержимое после лексического анализа для разбора.
Функция выполнения должна производить операцию над разобранным содержимым.
Функция lint должна сообщать о любых ошибках в разобранном содержимом.

::

    python early:
        def parse_smartline(lexer):
            who = lexer.simple_expression()
            what = lexer.rest()
            return (who, what)

        def execute_smartline(parsed_object):
            who, what = parsed_object
            renpy.say(eval(who), what)

        def lint_smartline(parsed_object):
            who, what = parsed_object
            try:
                eval(who)
            except Exception:
                renpy.error("Character not defined: {}".format(who))

            tte = renpy.check_text_tags(what)
            if tte:
                renpy.error(tte)

        renpy.register_statement(
            "line",
            parse=parse_smartline,
            execute=execute_smartline,
            lint=lint_smartline,
        )

Объект Lexer
~~~~~~~~~~~~

Функция парсинга пользовательского оператора принимает экземпляр объекта Lexer.

.. class:: Lexer

    .. property:: init

        Имеет значение true, если лексер находится внутри оператора init, иначе false.

    .. property:: init_priority

        Если init истинно, это приоритет блока init.

    .. method:: error(msg)

        :param str msg: Сообщение для добавления в список обнаруженных ошибок парсинга.

        Добавляет `msg` (с текущей позицией) в список обнаруженных
        ошибок парсинга. Это прерывает разбор текущего оператора,
        но не препятствует дальнейшему парсингу.

    .. method:: require(thing, name=None)

        Пытается разобрать `thing` и сообщает об ошибке, если это невозможно.

        Если `thing` — это строка, пытается разобрать её с помощью :func:`match`.

        В противном случае, thing должен быть другим методом этого объекта лексера, который
        вызывается без аргументов.

        Если `name` не указан, в сообщении будет использоваться имя метода
        (или `thing`, если это строка), иначе будет использоваться `name`.

    .. method:: eol()

        :return: True, если лексер находится в конце строки, иначе False.
        :rtype: bool

    .. method:: expect_eol()

        Если не в конце строки, вызывает ошибку.

    .. method:: expect_noblock(stmt)

        Вызывается, чтобы указать, что этот оператор не ожидает блока.
        Если блок найден, вызывает ошибку. `stmt` должен быть строкой,
        он будет добавлен в сообщение об ошибке.

    .. method:: expect_block(stmt)

        Вызывается, чтобы указать, что оператор требует наличия непустого
        блока. `stmt` должен быть строкой, он будет добавлен
        в сообщение об ошибке.

    .. method:: has_block()

        :return: True, если у текущей строки есть непустой блок, иначе False.
        :rtype: bool

    .. method:: match(re)

        Сопоставляет произвольную строку с регулярным выражением.

        Все операторы в лексере, которые выполняют сопоставление, реализованы
        через эту функцию. Они сначала пропускают пробельные символы, а затем пытаются
        выполнить сопоставление со строкой. Если сопоставление успешно, возвращается
        сопоставленный текст. В противном случае возвращается None, а состояние лексера
        остаётся неизменным.

    .. method:: keyword(s)

        Сопоставляет `s` как ключевое слово.

    .. method:: name()

        Сопоставляет имя. Это не сопоставляет встроенные ключевые слова.

    .. method:: word()

        :return: Текст сопоставленного слова.
        :rtype: str

        Сопоставляет любое слово, включая ключевые слова.

    .. method:: image_name_component()

        Сопоставляет компонент имени изображения. В отличие от слова, компонент
        имени изображения может начинаться с цифры.

    .. method:: string()

        Сопоставляет строку Ren'Py.

    .. method:: integer()

        :return: Строка, содержащая найденное целое число.
        :rtype: str

        Сопоставляет целое число.

    .. method:: float()

        :return: Строка, содержащая найденное число с плавающей точкой.
        :rtype: str

        Сопоставляет число с плавающей точкой.

    .. method:: label_name(declare=False)

        Сопоставляет имя метки, абсолютное или относительное. Если `declare`
        истинно, то устанавливается глобальное имя метки. (Обратите внимание, что это
        не объявляет метку на самом деле — оператор должен сделать это,
        вернув её из функции `label`.)

    .. method:: simple_expression()

        Сопоставляет простое выражение Python, возвращая его в виде строки.
        Это часто используется, когда ожидается имя переменной.
        Не рекомендуется изменять результат. Правильным действием будет
        вычислить результат в будущем.

    .. method:: delimited_python(delim)

        Сопоставляет выражение Python, которое заканчивается `delim`, например, ':'.
        Это часто используется, когда ожидается условие до разделителя.
        Не рекомендуется изменять результат. Правильным действием будет
        вычислить результат в будущем. Вызывает ошибку, если
        конец строки достигается до разделителя.

    .. method:: arguments()

        Этот метод должен вызываться перед скобками со списком аргументов.
        Если они не указаны, возвращает None, в противном случае
        возвращает объект, представляющий аргументы вызова
        функции. Этот объект имеет метод ``evaluate``, который
        принимает необязательный словарь `scope` и возвращает кортеж,
        в котором первый компонент — это кортеж позиционных аргументов,
        а второй — словарь именованных аргументов.

    .. method:: rest()

        Пропускает пробельные символы, затем возвращает оставшуюся часть строки.

    .. method:: checkpoint()

        Возвращает непрозрачный объект, представляющий текущее состояние лексера.

    .. method:: revert(o)

        Когда `o` — это объект, возвращенный из checkpoint(), возвращает состояние
        лексера к тому, каким оно было при вызове checkpoint(). (Это
        используется для отката.)

    .. method:: subblock_lexer()

        :return: Lexer для блока, связанного с текущей строкой.

    .. method:: advance()

        В лексере подблока переходит к следующей строке. Этот метод должен быть вызван
        перед первой строкой, чтобы можно было разобрать первую строку. Возвращает True,
        если мы успешно перешли к строке в блоке, или False,
        если мы вышли за пределы всех строк в блоке.

    .. method:: renpy_statement()

        При вызове этот метод разбирает текущую строку как оператор скрипта Ren'Py,
        вызывая ошибку, если это невозможно. Метод возвращает
        непрозрачный объект, который может быть возвращён из функции `next`, переданной в
        :func:`renpy.register_statement`, или передан
        в :func:`renpy.jump` или :func:`renpy.call`. Этот объект не следует
        хранить, кроме как в качестве части результата парсинга оператора.

        Когда оператор, возвращённый этим методом, завершается, управление
        передаётся оператору, следующему за пользовательским оператором.
        (Это может быть оператор, созданный с помощью post_execute).

    .. method:: renpy_block(empty=False)

        Разбирает все оставшиеся строки в текущем блоке
        как скрипт Ren'Py и возвращает SubParse, соответствующий
        первому оператору в блоке. Блок связывается таким образом,
        что все операторы в нём выполняются, а затем управление
        передаётся оператору, следующему за этим пользовательским оператором.

        Обратите внимание, что это разбирает текущий блок. В более вероятном
        случае, когда вы хотите разобрать подблок текущего
        оператора, правильный способ сделать это::

            def mystatement_parse(l):

                l.require(':')
                l.expect_eol()
                l.expect_block("mystatement")

                child = l.subblock_lexer().renpy_block()

                return { "child" : child }

        `empty`
            Если True, позволяет разбирать пустой блок. (Пустой блок
            эквивалентен блоку с одним оператором ``pass``.)

            Если False, пустой блок вызывает ошибку.

    .. method:: catch_error()

        Это декоратор контекста, используемый совместно с оператором with,
        который перехватывает и сообщает об ошибках лексера внутри своего
        блока контекста, а затем продолжает выполнение после блока.

        Вот пример того, как его можно использовать для сообщения о нескольких ошибках
        в одном подблоке. ::

            def mystatement_parse(l):

                l.require(':')
                l.expect_eol()
                l.expect_block("mystatement")

                strings = [ ]
                ll = l.subblock_lexer()

                while ll.advance():
                    with ll.catch_error():
                        strings.append(ll.require(ll.string))
                        ll.expect_noblock("string inside mystatement")
                        ll.expect_eol()

                return { "strings" : strings }
