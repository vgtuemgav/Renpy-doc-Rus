Действия
========

Многие из отображаемых объектов, созданных на экранном языке, принимают действия
в качестве аргументов. Действие — это одна из трёх вещей:

* Вызываемый Python-объект (например, функция или связанный метод), который
  не принимает аргументов.
* Объект класса, который наследуется от класса Action.
* Список других Действий.

Преимущество наследования от класса Action заключается в том, что это позволяет
вам переопределить методы, которые определяют, когда кнопка должна быть
активной, и когда она выбрана.

.. class:: Action

    Чтобы определить новое действие, наследуйтесь от этого класса. Переопределите
    методы в этом классе, чтобы изменить поведение действия.

    .. method:: __call__(self)

        Этот метод вызывается, когда действие
        активируется. Во многих случаях возврат значения, отличного от None, из
        действия приведёт к завершению текущего взаимодействия.

        Этот метод должен быть переопределен, так как метод по умолчанию
        вызовет NotImplementedError (и, следовательно, вызовет
        ошибку в Ren'Py).

    .. method:: get_sensitive(self)

        Вызывается для определения, должна ли кнопка с этим действием
        быть активной. Должен возвращать true, если кнопка активна.

        Обратите внимание, что __call__ может быть вызван, даже если этот метод возвращает False.

        Реализация по умолчанию возвращает True.

    .. method:: get_selected(self)

        Должен возвращать true, если кнопка должна отображаться как
        выбранная, и false в противном случае.

        Реализация по умолчанию возвращает False.

    .. method:: get_tooltip(self)

        Получает всплывающую подсказку по умолчанию для этой кнопки, если конкретная
        подсказка не назначена. Должен возвращать значение подсказки,
        или None, если подсказка не известна.

        По умолчанию возвращает None.

    .. method:: periodic(self, st)

        Этот метод вызывается один раз в начале каждого взаимодействия,
        а затем вызывается периодически. Если он возвращает
        число, он будет вызван до истечения этого количества секунд, но
        может быть вызван и раньше.

        Основное применение этого метода — вызов
        :func:`renpy.restart_interaction`, если значение
        get_selected или get_sensitive должно измениться.

        Он принимает один аргумент:

        `st`
            Количество секунд с момента, когда экран или отображаемый объект, с которым
            связано это действие, был впервые показан.

    .. method:: unhovered(self)

        Когда действие используется как параметр `hovered` для кнопки (или
        аналогичного объекта), этот метод вызывается, когда объект теряет фокус.

    .. attribute:: alt

        Текст для преобразования в речь, используемый для кнопок, которые используют это действие, если
        у кнопки не установлено свойство :propref:`alt`. Это может
        быть строка в классе, в конструкторе, или это может быть
        свойство Python, которое возвращает строку.

Чтобы запустить действие из Python, используйте :func:`renpy.run`.

BarValues
=========

При создании bar, vbar или hotbar, объект BarValue может быть предоставлен в качестве
свойства `value`. Методы объекта BarValue вызываются для получения
adjustment и стилей.


.. class:: BarValue

    Чтобы определить новый BarValue, наследуйтесь от этого класса и переопределите
    некоторые из методов.

    .. method:: get_adjustment(self)

        Этот метод вызывается для получения объекта adjustment для
        шкалы. Он должен создать adjustment с помощью
        :func:`ui.adjustment`, а затем вернуть созданный таким образом объект.

        Этот метод должен быть переопределен, так как метод по умолчанию
        вызовет NotImplementedError (и, следовательно, вызовет ошибку в Ren'Py).

    .. method:: get_style(self)

        Используется для определения стиля шкал, которые используют это
        значение. Должен возвращать кортеж из двух имен стилей или
        объектов стилей. Первый используется для bar, а
        второй для vbar.

        По умолчанию возвращает ("bar", "vbar").

    .. method:: get_tooltip(self)

        Получает всплывающую подсказку по умолчанию для этой кнопки, если конкретная
        подсказка не назначена. Должен возвращать значение подсказки,
        или None, если подсказка не известна.

        По умолчанию возвращает None.

    .. method:: replaces(self, other)

        Вызывается, когда BarValue заменяет другой BarValue, например,
        когда экран обновляется. Может использоваться для обновления этого
        BarValue из другого. Вызывается перед get_adjustment.

        Обратите внимание, что `other` не обязательно того же типа, что и `self`.

    .. method:: periodic(self, st)

        Этот метод вызывается один раз в начале каждого взаимодействия. Если
        он возвращает количество секунд, он будет вызван до
        истечения этого количества секунд, но может быть вызван и раньше. Он
        вызывается после get_adjustment.

        Он может быть использован для обновления значения шкалы со временем, как
        это делает :func:`AnimatedValue`. Для этого get_adjustment должен
        сохранить adjustment, а periodic должен вызывать
        метод changed объекта adjustment.

    .. attribute:: alt

        Текст для преобразования в речь, используемый для шкал, которые используют этот BarValue, если
        у шкалы не установлено свойство :propref:`alt`. Это может
        быть строка в классе, в конструкторе, или это может быть
        свойство Python, которое возвращает строку.


InputValue
==========

При создании input, объект InputValue может быть предоставлен в качестве
свойства `value`. Методы объекта InputValue вызываются для
получения и установки текста, определения, является ли поле ввода редактируемым, и обработки
нажатия клавиши Enter.

.. class:: InputValue

    Чтобы определить новый InputValue, наследуйтесь от этого класса, переопределите
    некоторые или все методы, и установите значение
    поля default.

    .. attribute:: editable

        Если не true, полностью запрещает редактирование поля ввода.

    .. attribute:: default

        Если true, поле ввода может быть редактируемым по умолчанию. (То есть,
        ему может быть передан курсор при показе экрана.)

    .. method:: get_text(self)

        Возвращает текст поля ввода по умолчанию. Этот метод должен быть реализован.

    .. method:: set_text(self, s)

        Вызывается, когда текст поля ввода изменяется, с новым текстом.
        Этот метод должен быть реализован.

    .. method:: enter(self)

        Вызывается, когда пользователь нажимает Enter. Если этот метод возвращает значение, отличное от None,
        это значение возвращается из взаимодействия. Этот метод также может
        вызвать :exc:`renpy.IgnoreEvent`, чтобы игнорировать нажатие. В противном случае,
        нажатие Enter передается другим отображаемым объектам.

    Следующие действия доступны как методы на InputValue:

    .. method:: Enable()

        Возвращает действие, которое включает редактирование текста в поле ввода.

    .. method:: Disable()

        Возвращает действие, которое отключает редактирование текста в поле ввода.

    .. method:: Toggle()

        Возвращает действие, которое переключает режим редактирования текста в поле ввода.

Инструкции экранного языка, определяемые разработчиком
======================================================

Ren'Py поддерживает определение пользовательских инструкций экранного языка. Определяемые разработчиком инструкции
экранного языка являются обёртками для инструкции экранного языка :ref:`use <sl-use>`.
Позиционные аргументы остаются позиционными аргументами, свойства становятся именованными аргументами,
и если инструкция принимает блок, то и инструкция `use` также его принимает. Например,
пользовательская инструкция экранного языка::

    titledwindow "Test Window":
        icon "icon.png"

        text "This is a test."

становится::

    use titledwindow("Test Window", icon="icon.png"):
        text "This is a test."

Определяемые разработчиком инструкции экранного языка должны быть зарегистрированы в блоке ``python early``.
Более того, файл, содержащий определяемую разработчиком инструкцию, должен быть загружен раньше
любого файла, который её использует. Поскольку Ren'Py загружает файлы в порядке сортировки их путей по Unicode,
обычно имеет смысл добавлять префикс, такой как 01 или другое небольшое число, к имени любого файла,
регистрирующего пользовательскую инструкцию.

Определяемые разработчиком инструкции экранного языка регистрируются с помощью функции renpy.register_sl_statement:

.. include:: inc/custom_sl

В качестве примера инструкции экранного языка, определяемой разработчиком, вот
реализация инструкции ``titledwindow``, приведенной выше. Во-первых,
инструкция должна быть зарегистрирована в блоке ``python early`` в файле, который загружается
рано — имя вроде 01custom.rpy часто загружается достаточно рано. Вызов регистрации
выглядит так::


    python early:
        renpy.register_sl_statement("titledwindow", children=1).add_positional("title").add_property("icon").add_property("pos")

Затем мы определяем экран, который реализует пользовательскую инструкцию. Этот экран может быть определен в
любом файле. Один из таких экранов::

    screen titledwindow(title, icon=None, pos=(0, 0)):
        drag:
            pos pos

            frame:
                background "#00000080"

                has vbox

                hbox:
                    if icon is not None:
                        add icon

                    text title

                null height 15

                transclude


Когда используются большие группы свойств, такие как `add_property_group`, имеет смысл использовать
синтаксис \*\*properties с ключевым словом properties в каком-то месте. Например::

    screen titledwindow(title, icon=None, **properties):
        frame:
            # When background not in properties it will use it as default value.
            background "#00000080"

            properties properties

            has vbox

            hbox:
                if icon is not None:
                    add icon

                text title

            null height 15

            transclude

Пользовательские инструкции экранного языка поддерживают предложение ``as``, которое принимает имя переменной. Если оно присутствует,
этой переменной будет присвоено значение `main` в области видимости экрана.
