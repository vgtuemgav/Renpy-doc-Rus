Видео
=====

Ren'Py способен использовать FFmpeg (входит в комплект) для воспроизведения видео, используя
видеокодеки:

* AV1
* VP9
* VP8
* Theora
* MPEG-4 part 2 (включая Xvid и DivX)
* MPEG-2
* MPEG-1

и следующие аудиокодеки:

* Opus
* Vorbis
* MP3
* MP2
* FLAC
* PCM

в следующих форматах контейнеров:

* WebM
* Matroska
* Ogg
* AVI
* Различные виды MPEG-потоков.

(Обратите внимание, что использование некоторых из этих форматов может потребовать патентных лицензий.
В случае сомнений, и особенно для коммерческих игр, мы рекомендуем использовать
AV1, VP9, VP8 или Theora; Opus или Vorbis; и WebM, Matroska или Ogg.)

Видео может отображаться в полноэкранном режиме или как отображаемый объект. Полноэкранные видео более эффективны.
Видео в формате YUV444 не имеют аппаратного ускорения, вместо этого используйте YUV420 или YUV422.

Декодер видео в Ren'Py не поддерживает видео с альфа-каналами, но параметр
`side_mask` отображаемого объекта :func:`Movie` может быть использован для этой
цели. Вот пример использования ffmpeg для создания файла webm с
маской side-by-side из файла mov с альфа-каналом. ::

    ffmpeg -i original.mov -filter:v alphaextract mask.mov
    ffmpeg -i original.mov -i mask.mov -filter_complex "hstack" -codec:v vp8 -crf 10 output.webm

Видео поддерживаются на веб-платформе, но список поддерживаемых кодеков отличается
от браузера к браузеру. Для кросс-браузерной совместимости (особенно для поддержки Safari)
самой эффективной комбинацией является H.264 с MP3 (или AAC) в файле MP4. Однако Ren'Py
не поддерживает декодирование H.264 (или AAC), поэтому эта комбинация может работать только на
веб-платформе.


Полноэкранные видео
-------------------

Самый простой и эффективный способ отобразить видео в полноэкранном режиме —
это использовать функцию :func:`renpy.movie_cutscene`. Эта функция отображает
видео в полноэкранном режиме, пока оно не закончится или игрок не нажмёт, чтобы
пропустить его. ::

    $ renpy.movie_cutscene("On_Your_Mark.webm")

Отображаемые видео и видео-спрайты
----------------------------------

Отображаемый объект Movie может быть использован для показа видео везде, где Ren'Py
может показать отображаемый объект. Например, видео может быть показано в качестве
фона экрана меню или как фон в игре.

Отображаемый объект Movie также может быть использован для определения видео-спрайта, который
является спрайтом, основанным на двух видео. Основное видео предоставляет
цвет спрайта. Второе видео, маска, предоставляет альфа-
канал, где белый цвет — полная непрозрачность, а черный — полная прозрачность.

Видео, воспроизводимые с помощью отображаемого объекта Movie, автоматически зацикливаются.

Вот пример определения видео-спрайта::

    image eileen movie = Movie(play="eileen_movie.webm", side_mask=True)

Видео-спрайт может быть показан с помощью инструкции show, которая автоматически начинает
воспроизведение видео. Воспроизведение будет автоматически остановлено, когда отображаемый объект будет скрыт. ::

    show eileen movie

    e "I'm feeling quite animated today."

    hide eileen

    e "But there's no point on wasting energy when I'm not around."

Отображаемый объект Movie также может быть использован как часть экрана, при условии, что он создан
на этапе init (например, как часть инструкции image). ::


    image main_menu = Movie(play="main_menu.ogv")

    screen main_menu:
        add "main_menu"
        textbutton "Start" action Start() xalign 0.5 yalign 0.5

На экране можно одновременно отображать несколько отображаемых видео или спрайтов,
в зависимости от производительности системы, при условии, что все они имеют одинаковую частоту кадров.
Поведение Ren'Py при отображении видео с разной частотой кадров
не определено, но, скорее всего, будет включать значительное количество пропущенных кадров.
