Переменные хранилища

В Ren'Py есть ряд переменных хранилища, которые управляют его функциями. Переменные хранилища могут быть изменены в любое время. Если переменная хранилища изменяется после запуска игры, она будет сохраняться и загружаться системой сохранений и откатываться при откате (rollback).

adv = Character(...)

Это шаблонный персонаж ADV-режима и тип персонажа по умолчанию, который используется при вызове Character().

_autosave = True

Эту переменную можно установить в False, чтобы отключить автосохранение.

_confirm_quit = True

Определяет, запрашивает ли игра подтверждение при выходе. Устанавливается в False во время сплэш-экрана и игнорируется в главном меню.

_constant

Если установлено в true в хранилище, указывает, что хранилище является константой. См. Константные хранилища.

default_mouse

По умолчанию не определено. Если определено, и если config.mouse установлено при запуске игры, это ключ, который используется для поиска курсора мыши, когда текущий курсор не существует или является курсором по умолчанию. Используется config.mouse и MouseDisplayable().

См. Пользовательские курсоры мыши для получения дополнительной информации.

_dismiss_pause = True

Если True, игрок может пропускать паузы и переходы.

_game_menu_screen = "save"

Это экран, который отображается при входе в игровое меню без выбора конкретного экрана. (Например, при щелчке правой кнопкой мыши, нажатии Escape или когда ShowMenu() вызывается без аргумента.) Если None, вход в игровое меню запрещён.

Устанавливается в None в начале сплэш-экрана и восстанавливает своё исходное значение по его завершении.

_greedy_rollback = True

Определяет, выполняет ли игра жадный откат (greedy rollback) после загрузки. Жадный откат откатится до состояния сразу после последнего оператора, с которым взаимодействовал игрок, а не до состояния прямо перед оператором, на котором игра находилась во время загрузки.

_history = True

Если true, Ren'Py будет записывать историю диалогов при показе реплики. (Обратите внимание, что config.history_length также должна быть установлена.)

_history_list = [ ]

Это список объектов истории, соответствующий каждой строке истории от самой старой до самой новой. См. раздел История для получения дополнительной информации.

_ignore_action = None

Когда это не None, это действие, которое выполняется после нажатия Ignore на экране обработки ошибок. Обычно это действие Jump(), чтобы перевести игру в место, где можно восстановиться после ошибки. Если None, управление переходит к следующему оператору Ren'Py.

main_menu = False

Ren'Py устанавливает эту переменную в True, находясь в главном меню. Это можно использовать для разного отображения экранов в главном меню.

_menu = False

Ren'Py устанавливает эту переменную в True при входе в контекст главного или игрового меню.

menu = renpy.display_menu

Функция, которая вызывается для отображения внутриигрового меню. Она должна принимать те же аргументы, что и renpy.display_menu(), и передавать неизвестные именованные аргументы без изменений. Присвоение nvl_menu() этой переменной отобразит меню NVL-режима.

mouse_visible = True

Управляет видимостью мыши. Автоматически устанавливается в true при входе в стандартные игровые меню.

name_only = Character(...)

Это шаблонный персонаж, который используется, когда в операторе say в качестве имени персонажа указана строка. Это:

"Eileen" "Hello, world."

эквивалентно:

$ temp_char = Character("Eileen", kind=name_only)
temp_char "Hello, world."

за исключением того, что переменная temp_char не используется.

narrator = Character(...)

Это персонаж, который произносит повествование (операторы say, в которых не указан персонаж или его имя). Это:

"Hello, world."

эквивалентно:

narrator "Hello, world."
_quit_slot = None

Если не None, это должна быть строка, указывающая имя слота файла. Когда Ren'Py завершает работу, игра будет сохранена в этот слот.

_rollback = True

Управляет, разрешён ли откат (rollback).

say : Callable

Функция, которую Ren'Py вызывает для отображения диалога, когда вместо говорящего персонажа используется строка:

define e = Character("Eileen", who_color="#0f0")

label start:
    "Eileen" "My name is Eileen." # вызовет функцию say
    e "I like trains !" # не вызовет функцию say

Эта функция должна иметь ту же сигнатуру, что и renpy.say(). Она не должна вызывать renpy.say(), а скорее использовать другие эквиваленты оператора say.

Редко приходится вызывать эту функцию напрямую, так как можно просто вызвать персонажа с диалогом. Эта переменная в основном существует для переопределения, как способ перехвата оператора say.

save_name = ""

Имя сохранения, которое включается в сохранения. Также используется временной шкалой Steam, если config.automatic_steam_timeline имеет значение true, и лучше всего рассматривать его как название главы.

_scene_show_hide_transition = None

Если не None, это переход, который будет выполнен с помощью оператора with после серии операторов scene, show и hide, за которыми не следует оператор with или оконный переход.

См. также

Автоматические переходы после Scene, Show и Hide

_screenshot_pattern = None

Если не None, эта строка используется вместо config.screenshot_pattern для определения имени файла скриншота. Пожалуйста, обратитесь к документации по этой переменной для формата строки.

_skipping = True

Управляет, разрешён ли пропуск.

_version = ...

Устанавливается в config.version при запуске новой игры. Может использоваться меткой after_load или config.after_load_callbacks для определения того, какие обновления необходимо выполнить.

Это устанавливается только один раз, при первом запуске. После этого игра сама отвечает за обновление _version по мере необходимости.

_window = False

Устанавливается операторами window show и window hide и косвенно оператором window auto. Если true, окно диалога отображается во время недиалоговых операторов.

_window_auto = False

Устанавливается в true оператором window auto и в false операторами window show и window hide. Если true, происходит автоматическое поведение окна.

_window_subtitle = ''

Добавляется к config.window_title для создания заголовка игрового окна. Автоматически устанавливается в config.menu_window_subtitle во время нахождения в игровом меню.
